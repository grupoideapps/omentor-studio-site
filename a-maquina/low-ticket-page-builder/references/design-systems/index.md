# Design System Catalog

Use este catálogo para escolher o `design system` da página antes de escrever logo, assets e direção visual.

## Ordem de decisão

1. Escolher um `design system` existente do catálogo.
2. Sintetizar um novo `design system` a partir de referência enviada pelo usuário.
3. Criar uma variação derivada de um sistema existente quando a referência pedir mistura controlada.

## Como escolher

- Use `Meta` quando a página precisar parecer produto, vitrine, hardware, showcase ou sistema mais visual.
- Use `Notion` quando a página precisar parecer ferramenta, documento, template, biblioteca ou produto de organização.
- Use qualquer outro sistema do catálogo quando a referência de marca ou produto combinar mais com a sua lógica.
- Use `novo a partir de referência` quando o usuário colar uma referência visual, enviar uma captura de tela, ou pedir um sistema próprio sem depender dos exemplos existentes.

## O que a skill deve extrair

Para qualquer sistema, capture sempre:

- tema e atmosfera
- paleta e papéis das cores
- tipografia e regras de escala
- botões, cards, inputs e navegação
- radius, sombras, bordas e superfícies
- grid, spacing e container
- comportamento responsivo
- regras de do / don't
- linguagem de imagem e motion, quando existir

## Regra de uso

Este catálogo é a fonte de decisão. O sistema escolhido precisa ser registrado no contexto de trabalho antes de criar:

- logo
- imagens e mockups
- prompts de assets
- camada visual do HTML
